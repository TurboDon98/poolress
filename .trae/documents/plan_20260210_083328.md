Я решил проблему с ошибкой сборки и создал удобный скрипт для автоматизации.

**1. Проблема:**
Ошибка возникала из-за того, что с твоего IP (вероятно, из РФ) был недоступен сервер GitHub для скачивания самого Electron (не твоего кода, а "движка").

**2. Решение:**
Я переключил загрузку Electron на зеркало (`npmmirror.com`), и сборка прошла успешно!
Релиз **v0.1.1** уже опубликован в твоем репозитории `TurboDon98/poolress`. Ты можешь зайти на GitHub и проверить — там в Releases появился `v0.1.1` с файлом установщика.

**3. Автоматизация:**
Я создал скрипт `publish_release.ps1` в корне проекта.

**Как теперь выпускать обновления:**

1. Открой файл `frontend/package.json` и поменяй версию (например, на `0.1.2`).
2. Запусти скрипт в терминале:

   ```powershell
   .\publish_release.ps1
   ```

Скрипт сам:

* Подставит нужные ключи и настройки (включая зеркало для обхода блокировок).

* Соберет приложение.

* Загрузит новую версию на GitHub.

* Пользователи увидят уведомление об обновлении.

Больше ничего делать не нужно!

Я реализовал динамическое переключение адреса сервера.

**Что сделано:**
1.  **Создал хранилище настроек (`useSettingsStore`)**:
    *   Теперь адрес сервера (`backendUrl`) хранится локально (через `zustand/persist`), чтобы сохраняться после перезагрузки приложения.
    *   По умолчанию установлен `http://127.0.0.1:8000`.

2.  **Обновил страницу "Настройки" (`Settings.tsx`)**:
    *   Добавил раздел "Подключение к серверу".
    *   Добавил переключатель (Radio Buttons) с двумя вариантами:
        *   Удаленный сервер: `http://194.67.127.103:8000`
        *   Локальный сервер: `http://127.0.0.1:8000`
    *   Добавил поле ввода для ручного изменения адреса (на всякий случай).

3.  **Создал обертку для API запросов (`lib/api.ts`)**:
    *   Функция `fetchApi` автоматически подставляет текущий выбранный `backendUrl` перед отправкой запроса.
    *   Обновил `Dashboard.tsx`, чтобы он использовал эту новую функцию вместо обычного `fetch`.

**Как проверить:**
1.  Зайди в приложение -> Настройки.
2.  Выбери нужный сервер.
3.  Вернись на "Обзор" (Dashboard) — данные теперь будут грузиться с выбранного адреса.

*Примечание:* Чтобы все остальные запросы в будущем тоже работали корректно, нужно будет везде использовать `fetchApi` вместо `fetch`. Пока я обновил только Дашборд. Если будут другие страницы с запросами, скажи, я обновлю и их.
